# محرّر ثيمات متاجر داسم — ملف الإنجاز الكامل

> ملخّص شامل لما أُنجِز في صفحة الثيم بلوحة تحكّم متاجر داسم (`dasm-stores`).
> التاريخ: 2026-06-16 · الحالة: **المراحل الثلاث مكتملة ومنشورة على الإنتاج + الذكاء مُفعّل**.
> الريبو: `DASMe-9/dasm-stores` · المسار: `/dashboard/theme-editor`.
> وثائق مرافقة: `store-theme-editor.md` (معماري حيّ) + `store-theme-editor.html` (تصور بصري).

---

## ١. الفكرة بإيجاز

محرّر ثيمات على نمط Shopify لأصحاب متاجر داسم (أصحاب معارض غير مبرمجين). شاشة مقسومة:

- **جهة الكود/البلوكات:** لغة وسوم مبسّطة جاهزة، يعبّيها صاحب المتجر أو يحذف.
- **جهة المعاينة:** رسم فوري في المتصفح يعكس كل تعديل لحظياً + معاينة فعلية للمتجر المنشور.
- **مساعد ذكاء:** يصف صاحب المتجر بالعربية، والذكاء يولّد البلوكات.

مبدأ حاكم: **«لا متجر فارغ»** — كل متجر يُولد بثيم افتراضي أنيق، فحتى من لا يفتح المحرّر يحصل على واجهة راقية.

## ٢. القرارات المعمارية (محسومة في جلسة التصميم)

| # | القرار | السبب |
|---|--------|-------|
| ١ | لغة الثيم = **وسوم مبسّطة مخصّصة** (لا Liquid حقيقي ولا HTML خام) | الجمهور غير مبرمج + الأمان (لا حوض تنفيذ) |
| ٢ | المعاينة = **هجين**: رسم فوري بالمتصفح + iframe فعلي | سلاسة Shopify + صدق الواجهة النهائية |
| ٣ | بنية المتجر = **سطحان**: واجهة أنيقة + صفحة منتجات؛ تفاصيل المنتج قالب محمي | الواجهة تبيع الانطباع، المنتجات تبيع؛ حماية مسار الشراء |
| ٤ | **«لا متجر فارغ»**: ثيم افتراضي + استعادة افتراضي | لا واجهات نصف مبنية تشوّه المنصة |
| ٥ | الذكاء = **مدمج** يُخرج بلوكات صالحة فقط | رُفض اللصق الخارجي لأنه يعيد فتح XSS |

## ٣. لغة البلوكات (DSL)

وسوم ذاتية الإغلاق، بلوك لكل سطر:

```
{# الواجهة — عدّل أو احذف أي بلوك #}
<navbar logo links="الرئيسية, المنتجات, تواصل" sticky />
<banner text="شحن مجاني للطلبات فوق ٢٠٠ ر.س" />
<hero title="{{ store.name }}" subtitle="تشكيلة مختارة" cta="تسوّق الآن" style="aurora" />
<featured title="الأكثر مبيعاً" limit="4" />
<product-grid title="كل المنتجات" cols="3" sort="newest" limit="12" />
<footer about="{{ store.name }}" terms="الشروط والأحكام" social="whatsapp, instagram" />
```

**الأنواع (allowlist):** `navbar` · `banner` · `hero` · `richtext` · `featured` · `product-grid` · `footer`.
**البيانات الديناميكية:** `{{ store.name }}` + منتجات المتجر داخل `featured`/`product-grid`.
**جوهري:** البلوكات **بيانات لا كود** — الرسم جدول ثابت `BlockType → مكوّن React`. لا تقييم JS/HTML/Liquid.

## ٤. نموذج الأمان (دفاع متعمّق)

| طبقة | الآلية | الملف |
|------|--------|-------|
| لا مسار تنفيذ | البلوكات وصف لا كود؛ الرسم جدول ثابت | `BlockRenderer.tsx` |
| Allowlist صارم | أنواع/خصائص معروفة فقط؛ الباقي يُسقط | `schema.ts` |
| تنقية النص | حذف `<script>`/أي وسم/`on*=`/`javascript:` + قصّ الطول | `sanitize.ts` |
| تحقّق قبل الحفظ | `validateBlock` على كل بلوك مُحلّل (coercion + clamp + oneOf) | `parse.ts` + `schema.ts` |
| عزل النطاق | الواجهة على `stores.dasm.com.sa`، بلا كوكيز مشتركة مع المنصة الأم | بنية النطاقات |
| لا وصول API | الثيم يقرأ `{{ store.* }}` المحقونة فقط | تصميم الرموز |
| **أمان الذكاء** | مخرجات النموذج تمرّ بنفس `parseBlocks → serializeBlocks` | `api/theme/generate.ts` |

**أسوأ احتمال:** مطوّر محترف يلصق كوداً خبيثاً ← يُنقّى/يُسقط ← ثيم قبيح في متجره هو فقط. لا يصل المنصة ولا متاجر أخرى ولا الخادم.

## ٦. التخزين

الوثيقة تُحفظ **إضافياً** داخل `store.theme_config.editor` (نفس مسار `PUT /my-store`)، دون كسر اكتشاف القوالب القديم (`preset_id`/`palette`):

```json
{
  "preset_id": "...",
  "editor": {
    "version": 2,
    "surfaces": {
      "landing": "<navbar/>\n<hero/>...",
      "products": "<navbar/>\n<product-grid/>..."
    }
  }
}
```

`readBlockDocument` يُرحّل شكل م١ `{version:1, source}` إلى `surfaces.landing` تلقائياً (توافق خلفي كامل).

## ٧. المراحل الثلاث

### المرحلة ١ — الأساس (PR #174 · `0f685d9`)
محرّك البلوكات + الشاشتان + الرسم الفوري + لوحة الإدراج + استعادة الافتراضي + الحفظ.
المسار `/dashboard/theme-editor` (إضافي — لم يمسّ معرض القوالب المحمي).

### المرحلة ٢ — تجربة Shopify (PR #175 · `35dece8`)
- **سطحان مستقلّان** (الواجهة + صفحة المنتجات) مع مبدّل + ترحيل من م١.
- **تحرير بصري** (`BlockList`): سحب/أسهم/حذف، متزامن مع الكود عبر `parse → serialize`؛ `navbar`/`footer` غير قابلة للحذف.
- **منتجات حقيقية** في المعاينة عبر `sellerApi.getProducts` + fallback للعيّنات.
- **معاينة فعلية** داخل `iframe` (`sandbox`) على `storefrontUrl(preview)`.

### المرحلة ٤ — البنّاء البصري «كانفا لمتاجر داسم» (PRs #178 · #179 · P4-C)
تحوّل من «محرّر ثيمات للمطوّر» إلى **بنّاء متجر بصري للمبتدئ**. التاجر يختار قالباً، يعدّل الأقسام بصرياً، وينشر بلا كود.

- **١٠ بلوكات جديدة:** `announcement` · `features` · `categories` · `image-banner` · `image-with-text` · `brands` · `testimonials` · `faq` · `newsletter` · `spacer` — كلها عبر نفس allowlist/sanitize/validate. نوع خاصية `url` آمن (`http(s)` فقط).
- **٦ قوالب جاهزة مجانية:** إكسسوارات سيارات · أزياء · إلكترونيات · تجميل · متجر عام · فاخر — كل قالب يملأ السطحين + تصميماً.
- **٣ أوضاع (المبتدئ افتراضي):** البنّاء البصري | مساعد الذكاء | المتقدّم (كود). الكود لم يعد أول ما يُعرض.
- **البنّاء البصري:** مكتبة أقسام مجمّعة (إضافة) + ترتيب + إخفاء/إظهار + حذف (navbar/footer محميّان) + تحرير حقول بصري من الـschema + إعادة للافتراضي + معاينة سطح/جوال.
- **ضوابط التصميم:** لون/خلفية/أزرار/حواف/عرض/بطاقة منتج/خط — كل قيمة enum ثابت (لا CSS حر). تحت `theme_config.editor.design`. إصدار المحرّر ← 3.
- **علم `hidden` عام** في الـDSL.
- **الربط بالواجهة العامة (هجين — البند #8):** `app/[slug]/page.tsx` و`products/page.tsx` يستهلكان الوثيقة المحفوظة عبر `StorefrontBlocks`: أقسام المحتوى تُرسم بنفس `BlockRenderer` (مطابق للمعاينة)، وبلوكات المنتجات تستخدم `ProductGrid` **الحقيقي التفاعلي** (روابط/سلة) فلا ارتداد على التسوّق. الترويسة/التذييل من chrome المتجر. **مبوّب:** فقط المتاجر التي خصّصت بالبنّاء (`theme_config.editor`)؛ غيرها دون تغيير. ترحيل v1/v2/v3 محفوظ.

### المرحلة ٣ — الذكاء المدمج (PR #176 · `c3bd613`)
- `pages/api/theme/generate.ts` يستدعي `claude-haiku-4-5` بـsystem prompt مُولّد من `describeBlocksForPrompt()` (يبقى متزامناً مع allowlist).
- **مخرجات النموذج تمرّ بنفس خطّ التحقّق** → لا سطح ثقة جديد.
- خانة الوصف في `SplitEditor` حيّة (إدخال + توليد + loading/خطأ)؛ الصفحة تُلحق البلوكات بالسطح النشط.
- حماية: Bearer + مهلة ٢٠ث + 503 لطيف إن غاب المفتاح.

## ٨. التحقّق

| المرحلة | الفحص |
|---------|-------|
| م١ | `tsc` نظيف + **١٢/١٢** اختبار سلوكي (allowlist/XSS/clamp/oneOf/round-trip) |
| م٢ | `tsc` نظيف + **١١/١١** اختبار (ترحيل v1→v2/حفظ السطوح/backfill/تحليل السطحين) |
| م٣ | `tsc` نظيف + **١١/١١** اختبار تنقية مخرجات وهمية (script/وسوم مجهولة/onerror/حقن img/قيم غير صالحة) |
| الكل | فحوص Vercel pass · دمج squash · **نشر إنتاج success** |

> ملاحظة صدق: التجربة التفاعلية المُصادَقة (login + backend + توليد فعلي) تُؤكَّد على staging بحساب متجر حقيقي.

## ٩. تفعيل الذكاء (مُنجَز)

أُضيف `ANTHROPIC_API_KEY` على مشروع Vercel لـ`dasm-stores` (production + preview + development، مشفّر) وأُعيد نشر الإنتاج بنجاح. **زر «توليد» فعّال الآن** على الموقع الرسمي.

---

## ١٠. كيف يُستخدم

1. لوحة تحكّم المتجر → **«المحرّر المتقدّم»** (من معرض القوالب أو `/dashboard/theme-editor`).
2. بدّل بين **الواجهة** و**صفحة المنتجات**.
3. حرّر بالكود أو بالتبويب **البصري** (سحب/حذف)، أو صف بالعربية واضغط **توليد**.
4. عاين فورياً، أو افتح **المتجر الفعلي**.
5. **حفظ التصميم** — ينعكس على الواجهة خلال دقيقة.

## ١١. المتبقّي (مستقبلي اختياري)

- تأكيد e2e تفاعلي على staging بحساب حقيقي.
- ربط البلوكات بمحرّك عرض المتجر العام (إن لزم تطابق ١٠٠٪ بين معاينة البلوكات والواجهة المنشورة).
- توسعة الرموز الديناميكية (`{{ store.* }}`) وأنواع بلوكات إضافية حسب الحاجة.
