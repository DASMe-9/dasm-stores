# W28 competitor follow-ups — 2026-06-14 to 2026-06-15

This file consolidates the useful follow-up findings from draft guardian PRs
#167, #168, #169, and #170 without replacing the existing W28 report.

## Consolidated findings

- `ProductTile` marketplace cart button still differs from the baseline shape:
  `rounded-xl` in code vs. circular `rounded-full` in the visual baseline.
- Store `ProductCard` needs a visible sold-out state in listing cards. Salla and
  Shopify patterns keep sold-out products understandable before the shopper opens
  the detail page.
- Storefront mobile shopping flow can benefit from a sticky mini cart bar when
  cart count is greater than zero. This should reuse the existing cart state and
  drawer rather than introducing a parallel cart flow.
- Salla-style lightweight cart confirmation is a separate backlog idea. It should
  not replace the existing cart drawer until the shopping flow is reviewed.
- `ProductCard` quick-add remains covered by the existing
  `docs/specs/product-card-quick-add-2026-06-13.md`; the later quick-add draft is
  treated as redundant to avoid conflicting implementation specs.
