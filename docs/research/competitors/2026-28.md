## Delta جديد — W28

### 1. Salla + Shopify: Sticky Product Bar للموبايل

**المصدر:**
- Salla theme-raed: mintlify docs تُصرّح بـ CSS لـ sticky product bar بـ `position: fixed; bottom: 0` مع animations
- Shopify Dawn 2026: Sticky Add-to-Cart bar للموبايل (مُؤكَّد في مراجعات 2026)

| التفصيل | Salla | Shopify Dawn |
|---------|-------|--------------|
| الموضع | `fixed bottom-0` على الموبايل | `fixed bottom-0` على الموبايل |
| المحتوى | عدد المنتجات + زر «أضف للسلة» | عدد العناصر + المجموع + زر «عرض السلة» |
| حالة الظهور | تظهر فقط عند وجود منتجات في السلة | تظهر عند scroll لأسفل أو عند إضافة منتج |
| الـ z-index | فوق المحتوى، تحت modals | نفسه |
| الحركة | `slideInUp` animation | `translateY` transition |

**الانطباق على dasm-stores:** غائب تماماً. `StoreChrome.tsx` يحتوي CartDrawer وWhatsApp FAB فقط. `CartBadge.tsx` موجود في الهيدر لكن يختفي عند scroll. لا sticky bar.

**الملف المتأثر:** `components/store/StoreChrome.tsx` (لإضافة المكوّن الجديد)

### 2. Salla: Cart Confirmation Popup

**المصدر:** WebSearch (innovatrixinfotech.com: Salla vs Shopify 2026)

Salla يُظهر **popup تأكيد فوري** بعد إضافة منتج للسلة يعرض:
- صورة مصغرة للمنتج المضاف
- زر «متابعة التسوق» + زر «إتمام الشراء»
- إجمالي السلة الجديد

**الانطباق على dasm-stores:**
- `CartDrawer.tsx` يُفتح بشكل كامل عند الإضافة (يمكن تحسينه بـ toast/popup خفيف)
- جزئي — يُضاف للـ ideas-backlog، لا spec هذه الجولة

### 3. Salla: getProductBadge() — حالة نفاد المخزون في listing

**المصدر:** تحليل مصدر `product-card.js` (SallaApp/theme-raed)

Salla يُنفّذ `getProductBadge()` في JavaScript مكوّن البطاقة للكشف عن:
- المنتجات النافدة (`out_of_stock`) → overlay نصي "نفد المخزون"
- الكميات المحدودة (`limited`) → شارة "كمية محدودة"

**الانطباق على dasm-stores:**
- `ProductCard.tsx` لا يُظهر أي مؤشر للمنتج النافد في listing
- موثّق في ideas-backlog منذ 2026-06-12 — يُؤكّده الكود الحي من Salla
- مقيّد بـ: `StoreProductCard` type لا يحتوي `stock_quantity` → يتطلب تعديل API response
